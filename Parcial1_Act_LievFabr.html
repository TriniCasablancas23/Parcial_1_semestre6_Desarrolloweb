1. Tecnologías de Web APIs

Arquitectura & Estándares

Interconexión de Sistemas

Las APIs actúan como una capa intermedia que procesa transferencias de datos entre sistemas, permitiendo exponer funcionalidades de forma segura bajo protocolos definidos.

RESTful Design

Estilo arquitectónico introducido por Roy Fielding (2000).

  • Cliente-Servidor: Separación de interfaz y datos.
  • Sin Estado (Stateless): Peticiones independientes sin contexto en servidor.
  • Caché: Respuestas optimizadas para rendimiento.
  • Capas: Soporte para intermediarios y balanceadores.
SOAP (Simple Object Access Protocol)

Protocolo estricto basado en XML con alta seguridad industrial.

2. Arquitectura de Microservicios

Filosofía de Diseño

División de una aplicación en servicios pequeños, independientes y desplegables. Cada servicio gestiona una lógica de negocio única (Pagos, Inventario, etc.).

  • Escalabilidad: Crecimiento solo del módulo necesario.
  • Resiliencia: Un fallo en uno no detiene al resto.
  • Políglota: Libertad para usar diferentes lenguajes.

Desafíos Críticos

La descentralización introduce nuevas barreras técnicas que requieren herramientas avanzadas de gestión.

  • Complejidad: Requiere orquestadores (Kubernetes).
  • Consistencia: Difícil mantener datos sincronizados (Saga pattern).
  • Latencia: Mayor tiempo de respuesta por llamadas de red.

3. Desarrollo de una Web API

Las APIs son mecanismos que permiten a dos componentes de software comunicarse mediante protocolos definidos.

Definición de métodos HTTP: GET obtención, POST creación, PUT actualización y DELETE eliminación.

Uso de frameworks (Express, Flask), implementación de JWT, autenticación y versionamiento de la API.

4. Manejo de Contenedores

¿Qué es?

Unidad estándar que empaqueta código y dependencias. Comparten el kernel del SO, siendo más ligeros que las VMs.

Docker

Crea Imágenes (moldes inmutables) que se ejecutan como Contenedores portables en cualquier entorno.

Kubernetes

El orquestador. Gestiona Pods, escala automáticamente ante tráfico y asegura alta disponibilidad.

5. Cloud-Native Development

Modelos de Implementación
  • Pública: AWS, Azure (Compartida).
  • Privada: Recursos exclusivos (Seguridad).
  • Híbrida: Combinación flexible.
Capas de Servicio
  • IaaS: Infraestructura (EC2).
  • PaaS: Plataforma (Heroku).
  • SaaS: Software (Gmail).
